# Prompt 1.0 示例（反面教材）

## 示例1：模糊的需求

### ❌ Prompt

```
帮我写一个用户管理系统。
```

### 结果

AI生成的代码可能：
- 功能缺失（没有权限管理）
- 结构混乱（所有代码在一个文件）
- 不符合你的技术栈（用Python而不是TypeScript）
- 没有错误处理
- 没有测试

---

## 示例2：过于简短

### ❌ Prompt

```
实现一个消息处理器。
```

### 结果

AI不知道：
- 要处理什么类型的消息？
- 处理完要做什么？
- 需要持久化吗？
- 性能要求高吗？
- 需要支持并发吗？

生成的代码很可能不符合需求。

---

## 示例3：混合多个需求

### ❌ Prompt

```
帮我写一个系统，要有用户管理、订单管理、支付功能，
还要支持多语言、多货币、实时通知，最好还能导出报表。
```

### 结果

- AI会随机选择实现哪些功能
- 代码复杂度失控
- 各部分之间耦合严重
- 难以测试和维护

---

## 示例4：缺乏约束

### ❌ Prompt

```
用Node.js写一个API服务器。
```

### 结果

- 不知道用什么框架（Express？Fastify？NestJS？）
- 不知道数据库（MongoDB？PostgreSQL？）
- 不知道是否需要认证
- 不知道API风格（REST？GraphQL？）

---

## 示例5：要求"完美"

### ❌ Prompt

```
帮我写一个完美的电商系统，要高性能、高可用、易扩展、易维护。
```

### 结果

- "完美"是主观概念，AI无法准确定义
- 生成的代码可能过度设计
- 包含大量不必要的抽象
- 学习曲线陡峭

---

## 总结：Prompt 1.0的问题

| 问题 | 后果 |
|------|------|
| 需求模糊 | AI随意发挥，结果不符合预期 |
| 过于简短 | 大量假设，导致偏差 |
| 混合需求 | 复杂度高，难以维护 |
| 缺乏约束 | 技术选型随机 |
| 要求"完美" | 过度设计，难以使用 |

**核心问题：AI不知道你的"设计意图"**
