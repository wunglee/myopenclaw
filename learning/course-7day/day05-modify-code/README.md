# Day 5 - 改代码：修改OpenClaw的一个简单功能

> 动手实践：用模型思维修改OpenClaw源码，提交你的第一个PR

---

## 课程目标

- **认知目标**：理解OpenClaw的代码结构
- **技能目标**：掌握用模型思维阅读和修改代码
- **行动目标**：成功修改一个OpenClaw功能并提交PR

---

## 课程结构（90分钟）

| 阶段 | 时长 | 内容 | 形式 |
|------|------|------|------|
| 准备 | 15分钟 | Clone代码、环境配置 | 实操 |
| 探索 | 20分钟 | 用模型思维阅读代码 | 分析 |
| 定位 | 15分钟 | 找到要修改的功能点 | 实操 |
| 修改 | 25分钟 | 动手修改代码 | 实操 |
| 提交 | 15分钟 | 测试、提交PR | 实操 |

## 核心任务

### 任务选择（三选一）

| 难度 | 任务 | 描述 |
|------|------|------|
| ⭐ 简单 | 添加日志 | 在消息处理流程中添加日志输出 |

| ⭐⭐⭐ 困难 | 添加新命令 | 实现一个新的斜杠命令 |

**建议**：第一次选择"简单"或"中等"难度
